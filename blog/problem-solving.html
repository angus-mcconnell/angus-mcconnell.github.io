Problem Solving

Why are Problem Solving Skills Important?

Problem solving is arguably the most important suite of "meta" skills that any programmer can develop. Why? Because regardless of the language being used, or the kind of program being developed, you can always count on running into unexpected issues. And the true measure of a programmer is how efficiently adnd effectively you can overcome them.

What are they?

Because the problems we encounter can be many and varied, so too must be the suite of skills used to solve them.


  • Pseudocode: Writing out the logic steps of what you want/need your code to do. Once you understand what you want your code to do, then you can work on figuring out the semantics of the language.

  • Trying something: Good old trial and error. It might not always be the most effecient way to solve a problem, but it can often yield unexpected results.

  • Rubber ducky method: Describing the code and what it does line by line to a real or imaginary person. Doing this can help spot incongruities between what the code is supposed to do and what it's written to do.

  • Reading error messages: Error messages might not always be easy to read or understand, but they are trying to tell you what's wrong.

  • Console.logging: console.log is very versatile function. You can use it throught out your code to report back on the value of variables before and after they are altered by functions to confirm that your code is working as intended.

  • Googling: The great equaliser. 9 times out of 10 someone has encountered your problem before, solved it, and then posted the solution somewhere online. When in doubt, Google it.

  • Asking your peers for help: Your peers are often working on the same problems as you. Even if they haven't solved it yet either you can at least work on it together.

  • Asking coaches for help: Same idea as behind asking your peers, but in this case they already have the answers because they made the questions. They should be able to point you in the right direction.

  • Improving your process with reflection: Taking the time to relfect on your problem solving process can p you get more effecient at the process. And if done when stuck on a problem it can help you gain insight to what you have, and more importantly, what you haven't tried.

Two Examples; Elegant and Frustrating

This week saw us running through a series of Javascript "katas". Both of these examples relate to the Gradebook exercise. At the start of the exercise we were given an array of student names, and an array of grade scores for the students. We were then asked to compile these two arrays into an object (gradebook) with the names action as the properties and scores as the property values.


The obvious solution is just to create the object by manually typing out the properties and values. But I figured that since they were already in arrays that I could get a little fancy with it and create a for loop that cycles through both arrays and applies them to the gradebook object. This didn't save me a great amount of time compared to doing it manually, but I liked getting to flex my brain muscles a little bit and automate a menial process. It was, I felt, an elegant solution.


Then at the end of the exercise, with the final task, I hit a wall. We were tasked with creating a function called enterScores and it was not even remotely clear to me what it was suposed to do. After taking a break to fetch a cup of tea I came back to the problem and applied the rubber ducky method to the testing code, trying to understand what it wanted the function to do.


My break through came when I realised that the test code was re-initialising the gradebook object that I'd already created, and was then using it and the student names, and grade score arrays as arguments for the enterScores function. It wanted me to create a function that automated the process of creating the gradebook object, just like I had already done at the start of the exercise unprompted. From there it was simply copying and pasting my already existing code into the function. Voila.
