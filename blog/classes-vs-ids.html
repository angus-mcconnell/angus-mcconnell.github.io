CSS Selectors

Classes and IDs are both forms of CSS Selectors. A selector is used to make applying CSS styling rules a simple and efficient process.


Without using a selector CSS rules have to be applied either inline for each individual element;


<h2 style="color: red; font-size 24pt;">An Alarming Title</h2>


Or as a blanket rule for all elements of the same type.


h2 {
  color: red;
  font-size 24pt;
}


Selectors can be used with a bit more finesse. By first declaring a class in your style sheet:


.redtext {
  color: red;
  font-size 24pt;
}


And then applying it to a specific element:


<h2 class="redtext">An Alarming Title</h2>

Class Selectors

Now the previous example I've just given is uses a class selector.


Classes are great because they can be applied to multiple elements in an html page and any single element can have multiple classes. This can save you a lot of work as your web pages become more elaborate and visually complex.

ID Selectors

So where do IDs fit into this?


Well the syntax for declaring and applying an ID Selector is similar enough:


#redtext {
  color: red;
  font-size 24pt;
}
...
<h2 id="redtext">An Alarming Title</h2>


Where an ID differs is in its uniqueness. Each ID can only be applied to a single element on your page, and each element can only have a single ID.

So How Do I Know Which One To Use?

At this stage it might begin to feel like the ID is a poor mans class. A class appears, at face value, to be vastly more versatile and efficient.


And for the most part this sentiment would be right. As far as application of style rules is concerned you will be better off using a class over an ID every time.


The true value and ID lies in uniqueness. Because only a single element on your page has that ID it means that you can create intra-page links that will jump directly to that element on the page.
